# TypeScript reference

The service-owned TypeScript API reference is generated by TypeDoc from `src/` only. Upstream `sysml-2ls` internals are intentionally excluded.

## Module index

| Module                                                                  | Responsibility                                                                      |
| ----------------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| [`app`](../../generated/typedoc/modules/app.md)                         | Fastify app factory, route wiring, HTTP errors, timeout behavior, effective limits. |
| [`contracts`](../../generated/typedoc/modules/contracts.md)             | Public DTOs and Zod request schemas.                                                |
| [`diagnostics`](../../generated/typedoc/modules/diagnostics.md)         | Langium/upstream diagnostic normalization into service DTOs.                        |
| [`limits`](../../generated/typedoc/modules/limits.md)                   | Service-owned request limit defaults and environment parsing.                       |
| [`sysml-validator`](../../generated/typedoc/modules/sysml_validator.md) | Adapter boundary to upstream `sysml-2ls`.                                           |
| [`uri`](../../generated/typedoc/modules/uri.md)                         | Request-local document URI generation and language inference.                       |
| [`version`](../../generated/typedoc/modules/version.md)                 | `/v1/version` metadata composition.                                                 |

## Generation command

```bash
pnpm run docs:generate:typedoc
```

TypeDoc output is committed under `docs/generated/typedoc`. If public TypeScript exports change, regenerate this output and include it in the PR.
